# Glossário

> **Para o Cliente:** Este glossário explica os termos técnicos usados na documentação de forma simples. Se você encontrar uma palavra que não entende, provavelmente ela está aqui!

---

## A

### Agente (Agent)
Um programa de computador que age de forma autônoma para realizar tarefas. No nosso caso, o **Agente SDR** é um programa que conversa com clientes pelo WhatsApp.

### Agno Framework
Biblioteca Python especializada em criar agentes de IA. É como um "kit de ferramentas" para construir assistentes inteligentes.

### API (Application Programming Interface)
"Interface de Programação". É como um "cardápio" que lista o que um sistema pode fazer e como pedir cada coisa. Outros sistemas usam a API para se comunicar.

### API Key
Uma "senha especial" que identifica quem está usando um serviço. Como um crachá que dá acesso a determinadas áreas.

### Async/Assíncrono
Forma de programação onde o sistema não precisa esperar uma tarefa terminar para começar outra. Como um chef que coloca algo no forno e vai preparar outro prato enquanto espera.

---

## B

### Backend
A parte do sistema que o usuário não vê - onde acontece todo o processamento. Como a cozinha de um restaurante.

### Backup
Cópia de segurança dos dados. Como guardar uma cópia xerox de documentos importantes.

### Bash
Um "terminal" onde você digita comandos para o computador executar. Como uma linha de comando.

---

## C

### Cache
Armazenamento temporário de dados para acesso rápido. Como deixar ingredientes pré-preparados na bancada para cozinhar mais rápido.

### Chatwoot
Plataforma de atendimento ao cliente que mostra as conversas em uma interface visual. Como um painel de controle para acompanhar os atendimentos.

### CLI (Command Line Interface)
Interface de linha de comando. Uma forma de interagir com o computador digitando texto em vez de clicar com o mouse.

### Cloud (Nuvem)
Servidores remotos acessados pela internet. Em vez de rodar no seu computador, o sistema roda em computadores da Amazon, Google, etc.

### Container
Uma "caixa" que contém todo o software necessário para rodar uma aplicação. Garante que funcione igual em qualquer lugar.

### CRM (Customer Relationship Management)
Sistema de gestão de relacionamento com cliente. Onde a equipe de vendas organiza e acompanha oportunidades de negócio.

### CRUD
Create, Read, Update, Delete (Criar, Ler, Atualizar, Deletar). As quatro operações básicas que se faz com dados.

---

## D

### Dashboard
Painel visual que mostra informações importantes de forma resumida. Como o painel de um carro.

### Deploy
O processo de colocar um sistema para funcionar em um servidor. Como "ligar" o sistema para produção.

### Docker
Tecnologia para criar e rodar containers. Empacota o sistema para funcionar igual em qualquer lugar.

### Domínio
O endereço de um site na internet. Por exemplo: `seleto.com.br`

---

## E

### E.164
Formato padrão internacional para números de telefone. Exemplo: `5511999999999` (código do país + DDD + número).

### Endpoint
Um "ponto de acesso" em uma API. Como uma porta específica em um prédio - cada uma leva a um lugar diferente.

### ENV (Environment Variables)
Variáveis de ambiente. Configurações que ficam fora do código, como senhas e chaves de API.

---

## F

### FastAPI
Framework Python para criar APIs web. O "esqueleto" que estrutura nosso sistema.

### Framework
Conjunto de ferramentas e padrões pré-definidos que facilitam criar software. Como um kit de construção.

### Frontend
A parte do sistema que o usuário vê e interage. Como o salão de um restaurante.

---

## G

### Git
Sistema de controle de versão. Guarda o histórico de todas as mudanças no código, como um "histórico de salvamentos".

### GPT (Generative Pre-trained Transformer)
Tipo de modelo de IA que gera texto. GPT-4o é a versão que usamos da OpenAI.

### Guardrails
"Cercas de proteção" - regras que limitam o que o sistema pode fazer. Por exemplo, não falar sobre preços.

---

## H

### Hash
Uma "impressão digital" de dados. Transforma qualquer texto em um código único de tamanho fixo.

### Health Check
Verificação de saúde. Um teste automático que confirma se o sistema está funcionando.

### HTTPS
Versão segura do HTTP (protocolo da web). Os dados são criptografados durante a transmissão.

---

## I

### IDE (Integrated Development Environment)
Programa para escrever código. Como um "Word" para programadores.

### Idempotente
Uma operação que pode ser repetida várias vezes sem mudar o resultado. Como apertar o botão do elevador - apertar 10 vezes não chama 10 elevadores.

### Integração
Conexão entre dois sistemas diferentes que precisam trocar informações.

---

## J

### JSON (JavaScript Object Notation)
Formato de texto para trocar dados entre sistemas. Fácil de ler tanto por humanos quanto por computadores.

### JWT (JSON Web Token)
Um "cartão de identidade digital" que sistemas usam para autenticar usuários.

---

## L

### Lead
Potencial cliente que demonstrou interesse em comprar. Alguém que pode virar uma venda.

### LGPD (Lei Geral de Proteção de Dados)
Lei brasileira que regula como empresas devem tratar dados pessoais.

### LLM (Large Language Model)
"Modelo de Linguagem Grande". IA treinada em muito texto que consegue entender e gerar linguagem natural. GPT-4 é um LLM.

### Log
Registro de eventos do sistema. Como um diário que anota tudo que acontece.

---

## M

### Middleware
Código que fica "no meio" entre uma requisição e a resposta. Pode adicionar funcionalidades como logging.

### Migration
Script que altera a estrutura do banco de dados. Como uma "receita" para criar ou modificar tabelas.

### Mock
Simulação de algo real para testes. Como um "boneco de treino" em vez de pessoa real.

---

## N

### Normalização
Padronizar dados para um formato consistente. Exemplo: telefone `(11) 99999-9999` vira `5511999999999`.

---

## O

### OpenAI
Empresa que criou o ChatGPT e fornece a IA que usamos no sistema.

---

## P

### Payload
Os dados enviados em uma requisição. O "conteúdo do pacote".

### PipeRun
CRM brasileiro que pode ser integrado ao sistema para gestão de vendas.

### PostgreSQL
Banco de dados relacional que o Supabase usa por baixo dos panos. Um dos bancos mais confiáveis do mundo.

### Produção
O ambiente onde o sistema roda "de verdade", atendendo clientes reais.

### Prompt
Instruções dadas para uma IA. O "pedido" que você faz para o modelo de linguagem.

### Python
Linguagem de programação que usamos no sistema. Conhecida por ser fácil de ler e muito usada em IA.

---

## Q

### Query
Uma pergunta feita ao banco de dados. Como pedir: "me dê todos os leads de São Paulo".

---

## R

### Rate Limiting
Limite de quantas requisições um sistema aceita por período. Previne sobrecarga.

### REST (Representational State Transfer)
Padrão de arquitetura para APIs. Define como sistemas devem se comunicar pela internet.

### Retry
Tentar novamente quando algo falha. Como discar de novo quando a ligação cai.

### RLS (Row Level Security)
Segurança em nível de linha no banco de dados. Cada usuário só vê os dados que tem permissão.

### Rollback
Voltar para uma versão anterior do sistema. Como usar "desfazer" no Word.

---

## S

### SDK (Software Development Kit)
Conjunto de ferramentas para desenvolver software para uma plataforma específica.

### SDR (Sales Development Representative)
Profissional de vendas responsável por qualificar leads. Nosso agente faz isso automaticamente.

### Singleton
Padrão onde só existe uma instância de algo no sistema. Como ter apenas um presidente por vez.

### SQL (Structured Query Language)
Linguagem para consultar bancos de dados. "SELECT * FROM leads" busca todos os leads.

### Staging
Ambiente de testes que imita a produção. Para testar antes de "ir ao vivo".

### Supabase
Plataforma que fornece banco de dados PostgreSQL na nuvem. Onde guardamos todos os dados.

### System Prompt
Instruções iniciais dadas ao modelo de IA. Define personalidade e regras de comportamento.

---

## T

### Temperatura (de Lead)
Classificação do potencial de compra: quente (pronto para comprar), morno (interessado), frio (apenas pesquisando).

### Timeout
Tempo limite para uma operação. Se demorar mais, é cancelada.

### Token
1. Unidade de texto para IA (palavras/partes de palavras)
2. Chave de autenticação (como uma senha)

### Troubleshooting
Processo de identificar e resolver problemas.

---

## U

### Upsell
Sugerir um produto melhor/mais caro. Exemplo: cliente quer FBM100, sugerimos FB300.

### URL (Uniform Resource Locator)
Endereço completo de algo na internet. Exemplo: `https://api.example.com/webhook`

### UUID (Universally Unique Identifier)
Identificador único universal. Um código como `123e4567-e89b-12d3-a456-426614174000`.

---

## V

### Validação
Verificar se dados estão corretos antes de usar. Como conferir se um CPF tem 11 dígitos.

### Variável de Ambiente
Configuração guardada fora do código. Definida no sistema operacional ou arquivo `.env`.

---

## W

### Webhook
Quando um sistema avisa outro que algo aconteceu. Exemplo: Z-API avisa nosso sistema quando chega mensagem.

### WhatsApp
Aplicativo de mensagens que nossos clientes usam para conversar com o agente.

### Whisper
Modelo da OpenAI para transcrição de áudio em texto.

---

## X

### XML (eXtensible Markup Language)
Formato de texto estruturado. Usamos para os prompts do agente.

### XXE (XML External Entity)
Tipo de ataque de segurança via XML. Nosso sistema está protegido contra isso.

---

## Z

### Z-API
Serviço brasileiro que conecta sistemas ao WhatsApp. A "ponte" entre nosso sistema e o WhatsApp.
